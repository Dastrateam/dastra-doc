# Bibliothèque

Elle constitue le **socle commun** sur lequel reposent les frameworks, les projets de conformité et la gestion des risques.

{% columns %}
{% column %}
Depuis la bibliothèque, vous pouvez créer, structurer et maintenir :

* des **frameworks** (référentiels de conformité),
* des **exigences**,
* des **contrôles**, éléments centraux reliant exigences, tests et risques,
* ainsi que les **risques** et **menaces** associés.
{% endcolumn %}

{% column %}

{% endcolumn %}
{% endcolumns %}

La bibliothèque permet une **approche transverse et réutilisable** de la conformité :\
un même contrôle peut couvrir plusieurs exigences, être partagé entre plusieurs frameworks et être évalué à l’aide de tests standardisés.\
Cette centralisation facilite la gouvernance, améliore la cohérence des référentiels et offre une vision globale grâce à des indicateurs et statistiques dédiés.
